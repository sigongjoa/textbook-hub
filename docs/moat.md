# Moat 분석 및 포지셔닝

> 경쟁자가 따라잡을 수 없는 구조적 우위가 무엇인가. 솔직한 평가.

## 가짜 moat (착각)

| 후보 | 실제 |
|---|---|
| 블록·Fork UX | 3개월이면 복제. GitHub 흉내는 누구나. |
| 한정 프로세스 카탈로그 | 문서화된 룰셋. OSS로 공개되는 순간 무력화. |
| LLM 활용 자체 | 상품화됨. OpenAI/Anthropic API 쓰면 누구나. |
| 공유 커뮤니티 | iScream·수만휘·수학사랑 카페가 이미 선점. 후발주자의 네트워크 효과는 환상. |
| Typst/CeTZ 렌더링 | 기술 우위는 되지만 moat 아님. |

이것들을 "우리 차별점"이라고 믿는 순간 전략 실패.

## 진짜 moat 후보 3개 (강도 순)

### ① 학생 × Lesson 매칭 데이터 flywheel (최강, 느림)
- **어떤 학생 프로필(학년·오답 유형·학습속도)에 어떤 Lesson + 프로세스 조합이 성적↑로 이어졌는가**
- 이 데이터는 쓸수록 추천 정확도 상승 → 유저 증가 → 데이터 증가
- 경쟁자가 처음부터 따라잡을 수 없는 유일한 **축적형 자산**
- **단점**: scale 전엔 0. 학원 20개·학생 500명 이상부터 작동.

### ② 한국 수학 콘텐츠 특화 검증 calibration (단기, 중간)
- math-verify 파이프라인 자체는 OSS 복제 가능 (6~12개월)
- 하지만 **한국 교재 idiom** — 서술형 채점 기준, 수능 기출 패턴, 학년별 허용 어휘 — 에 맞춘 룰셋·테스트케이스·fail 사례 데이터는 축적형
- 경쟁자가 OSS 복제해도 "한국 중3 서술형 95% 정확 검증" 같은 calibration은 수년 걸림
- 단기 방어선, 장기는 희석됨

### ③ 학원 운영 통합 (switching cost, 구조적)
- `EducationCoreArchitecture`에 이미 있는 자산: `class-log`, `homework`, `task-log`, `teach-note`, `img-gen`
- **학생관리 + 숙제배포 + 진도기록 + 교재생성**이 한 시스템에 엮이면 학원장 이탈 비용 폭증
- 교재만 공유하는 단일 기능 경쟁자는 이길 수 없는 구조적 lock-in
- **이게 실질적으로 가장 현실적인 moat** — 이미 재료가 있음

## 시간축 전략

| 기간 | moat | 실행 |
|---|---|---|
| 0~6개월 | 없음. 속도·실행력·초기 사용자 밀착만. | 학원 5개 pilot, hands-on 온보딩. |
| 6~18개월 | 검증 calibration + 학원 운영 통합 | `class-log`·`homework`와 통합. 교재만 파는 플랫폼 → "학원 OS" 포지션 이동. |
| 18개월+ | 데이터 flywheel | 개별화 추천 엔진. 이때부터 대기업도 따라잡기 어려움. |

## 경쟁 시나리오

### vs 대기업 (비상·천재·EBS·카카오)
자본·유통력 압도. 교재 콘텐츠 단독 승부로는 못 이김.
→ **학원 OS 통합**으로 우회. 대기업은 B2C 교재 유통이 본업이라 학원 내부 업무 통합은 후순위.

### vs 소규모 OSS 경쟁자
calibration·학원 통합에서 우위.

### vs 선생님 개인 유튜버/블로그
동일 레이어 아님. 경쟁 관계 없음 (오히려 콘텐츠 공급원).

## 핵심 결론

> **"교재 버전관리 플랫폼"으로 남으면 moat 없음. "학원 OS 안에 통합된 교재 모듈"로 재포지셔닝해야 방어 가능.**

현재 `EducationCoreArchitecture`의 `class-log` / `homework` / `teach-note` / `task-log` 자산이 이 포지셔닝을 가능하게 하는 **유일한 구조적 우위**. textbook-hub을 독립 SaaS로 분리하지 말고 기존 시스템의 **콘텐츠 레이어**로 설계하는 게 전략적으로 맞음.

## 포지셔닝 변경 요약

| 항목 | 이전 | 변경 |
|---|---|---|
| 제품 정체성 | 교재 공유 커뮤니티 웹 SaaS | 학원 OS 안의 교재 모듈 |
| 1차 타깃 | 선생님 전반 | 소형 학원장 (월매출 500~2000) |
| Fork 단위 | Block | Lesson (+ 내부 Block 편집) |
| MVP 범위 | 공유 커뮤니티 포함 | 개인 라이브러리 + 학생별 Fork만 |
| 수익 모델 | 미정 | 3-tier (Free/Paid/Private) + 70-30 분배 |
| AI 역할 | 미정 | 실행엔진 / 검증 / 추천 3-layer |
