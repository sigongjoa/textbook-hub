# 유즈케이스 정의

> **전제**: Fork 단위는 블록. 편집은 한정된 프로세스 안에서만.

## 액터 (Actor)

| 액터 | 설명 |
|---|---|
| **작성자 선생님 (Author)** | 블록을 만들어 올리는 사람 |
| **소비자 선생님 (Consumer)** | 남의 블록을 가져다 쓰는 사람 |
| **수정자 선생님 (Adapter)** | Fork 후 한정 프로세스로 변형하는 사람 — Consumer와 같은 사람이기도 함 |
| **시스템 (Platform)** | 플랫폼 자체. 검증·추천·렌더링 담당 |

> 한 선생님이 세 역할을 모두 수행할 수 있음. GitHub의 user가 owner·forker·contributor인 것과 동일.

---

## UC-01: 블록 업로드 (Author)

**목표**: 내가 쓴 개념 설명/문제를 커뮤니티에 올린다.

**플로우**:
1. Author가 블록 유형 선택 (개념·예제·문제·시각자료·풀이)
2. 블록 내용 작성 (에디터 또는 .typ 업로드)
3. 메타데이터 입력 (과목·학년·단원·난이도·태그)
4. 플랫폼이 자동 검증 (math-verify / prose-verify)
5. 검증 통과 시 공개

**산출**: 공개된 블록 1개 (버전 v1)

---

## UC-02: 블록 탐색 (Consumer)

**목표**: 내 수업에 쓸 만한 블록을 찾는다.

**플로우**:
1. Consumer가 단원·키워드로 검색
2. 플랫폼이 매칭 블록 리스트 제공
3. 블록 미리보기 (렌더된 형태로)
4. 블록 상세 페이지: 원본 + 파생 버전 트리 확인
5. "사용" 또는 "Fork" 선택

**변형**: 
- UC-02a: 교재/단원 통째로 탐색 후 거기서 블록만 뽑아가기
- UC-02b: 다른 선생님 프로필 따라가서 그 사람 블록 훑기

---

## UC-03: 블록 그대로 사용 (Consumer)

**목표**: 남의 블록을 내 교재에 그대로 삽입.

**플로우**:
1. Consumer가 블록 페이지에서 "내 교재에 담기"
2. 내 교재 중 삽입 위치 선택
3. 블록이 **참조 링크**로 삽입 (원본 블록 id 참조, 복사 아님)
4. 원본 작성자에게 "사용됨" 크레딧 카운트 +1

**핵심 결정사항**: 그대로 사용은 "참조"인가 "복사"인가?
- (A) 참조: 원본이 업데이트되면 내 교재도 자동 업데이트. 단점: 원본이 내려가면 깨짐.
- (B) 복사: 독립. 단점: 원본 개선이 전파 안 됨.
- (C) 참조 + pin(특정 버전 고정): 기본값 참조, 사용자가 특정 버전에 고정 가능.

**→ (C) 추천**. GitHub submodule + package lockfile과 유사한 철학.

---

## UC-04: 블록 Fork + 변형 (Adapter) ★ 핵심 유즈케이스

**목표**: 남의 블록을 가져와서 내 학생에 맞게 바꾼다.

**플로우**:
1. Adapter가 블록 페이지에서 "Fork"
2. 내 워크스페이스에 블록 복사본 생성 (parent 링크 유지)
3. **한정된 프로세스 중 하나 선택**:
   - `난이도 조정` (올림/내림)
   - `길이 조정` (줄임/늘림)
   - `수치 교체` (구체적 숫자만 다른 문제로)
   - `문맥 교체` (예: "사과"→"축구공")
   - `학년 변환` (예: 고1→중3)
   - `언어 변환` (설명 스타일: 격식↔친근)
   - `풀이 추가/제거`
   - `오답 선지 재생성` (MCQ)
4. 프로세스 실행 (AI 또는 수동 편집, 자유편집 아님)
5. 플랫폼이 자동 검증 (math-verify / prose-verify)
6. 변형본을 내 교재에 저장 or 커뮤니티에 공개

**핵심**: 편집은 **자유형이 아니라 정해진 오퍼레이션의 조합**. 이 제약이 제품의 핵심 차별점.

**결과물 관계**: fork된 블록은 원본과의 "변형 계보(lineage)"를 유지. 원본 작성자는 "내 블록에서 파생된 N개 버전" 볼 수 있음.

---

## UC-05: 블록 버전 업데이트 (Author)

**목표**: 이미 올린 블록을 개선한다.

**플로우**:
1. Author가 본인 블록에서 "새 버전"
2. UC-04와 동일한 한정 프로세스로 수정 (또는 제약 없이 전면 개편)
3. 새 버전 v2 공개
4. 플랫폼이 기존 사용자(참조 링크 가진 사람)에게 알림: "v2 나왔음. 반영할래?"

---

## UC-06: Fork 계보 보기 (모두)

**목표**: 블록의 파생 역사 시각화.

**플로우**:
1. 블록 상세 페이지 → "변형 트리"
2. 원본 → 파생본 → 파생의 파생 ... 트리로 표시
3. 각 파생본에 **어떤 프로세스가 적용됐는지** 라벨 표시 (예: `+난이도↑`, `+문맥교체`)

**가치**: "이 블록은 고1→중3 하향 변형이 가장 인기군" 같은 통계 가능.

---

## UC-07: 학생별 Fork (Adapter, 사적 사용)

**목표**: 같은 블록을 학생별로 다르게 변형해 배포.

**플로우**:
1. Adapter가 블록 Fork
2. 내 학생 목록에서 다중 선택 (예: 이루다, 김하진)
3. 학생별로 다른 프로세스 적용:
   - 이루다: `난이도↓` + `풀이 추가`
   - 김하진: `수치 교체` × 3회 (반복 연습)
4. 학생별 개인 교재에 삽입 (비공개)

**주의**: 이 경로는 비공개 + 학생 데이터 연동 → 단기 개인 학원용, 장기적으로 커뮤니티 확장 동력.

---

## UC-08: 블록 신고/수정 제안 (Consumer → Author)

**목표**: 오류 발견 시 원작자에게 피드백.

**플로우**:
1. Consumer가 블록에서 "이 부분 문제 있음" 
2. 제안 내용 작성 (오타·오답·더 나은 설명 등)
3. 플랫폼이 Author에게 알림
4. Author가 승인하면 v2로 반영 (Consumer에게 크레딧)
